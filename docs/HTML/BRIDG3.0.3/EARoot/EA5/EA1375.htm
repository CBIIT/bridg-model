<p>A name for a person, organization, place or thing. <br/></p><p><br/></p><p>Examples:   "Jim Bob Walton, Jr.", "Health Level Seven, Inc.", "Lake Tahoe", etc. An entity name may be as simple as a character string or may consist of several entity name parts, such as, "Jim", "Bob", "Walton", and "Jr.", "Health Level Seven" and "Inc.".<br/></p><p><br/></p><p>Entity names are essentially sequences of entity name parts, but add a "use" code and a valid time range for information about when the name was used and how to choose between multiple aliases that may be valid at the same point in time.<br/></p>

public Set(EntityNameUse)
  use
Notes: A set of codes advising a system or user which name in a set of names to select for a given purpose. <br />A name without specific use code might be a default name useful for any purpose, but a name with a specific use code would be preferred for that respective purpose. Names SHOULD not be collected without at least one use code, but names MAY exist without  use code, particularly for legacy data.<br /><br />If populated, the values contained in this attribute SHALL be taken from the HL7 EntityNameUse2 code system

Notes: The Entity Name with a standard ordering imposed on the parts. <br />The canonical form is primarily defined for the purposes of defining equality, and may differ from the socially accepted order for the name parts in various cultures around the world.<br />The canonical form contains all the part types except for the delimiters, in the following order:<br />1.    prefixes with qualifier title<br />2.    given names, with any prefixes and/or suffixes associated with the given names<br />3.    family names, with any prefixes and/or suffixes associated with the family names<br />4.    suffixes with qualifier title<br />Each list of part types SHALL be in the order of the original name.
