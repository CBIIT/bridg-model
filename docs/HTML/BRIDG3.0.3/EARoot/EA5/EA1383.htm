<p>A periodic or monotone sequence of values generated from a few parameters, rather than being enumerated. Used to specify regular sampling points for biosignals.<br/></p>

Attribute Details
public T
  head		 Range:0 to 1
Notes: The first item in this sequence. This is the start-value of the generated list.
public QTY
  increment		 Range:0 to 1
Notes: The difference between one value and its previous different value. <br /><br />For example: To generate the sequence (1; 4; 7; 10; 13; ...) the increment is 3; likewise to generate the sequence (1; 1; 4; 4; 7; 7; 10; 10; 13; 13; ...) the increment is also 3. The actual type QTY will be dictated by the type of T. The value of increment must be positive.
public Integer
  period		 Range:0 to 1
Notes: If not null or nullFlavored, specifies that the sequence alternates, i.e., after this many increments, the sequence item values roll over to start from the initial sequence item value. <br />Example:   The sequence (1; 2; 3; 1; 2; 3; 1; 2; 3; ...) has period 3; also the sequence (1; 1; 2; 2; 3; 3; 1; 1; 2; 2; 3; 3; ...) has period 3 too.<br />The period allows to repeatedly sample the same sample space. The "waveform" of this periodic generator is always a "saw", just like the x-function of your oscilloscope.
public Integer
  denominator		 Range:0 to 1
Notes: The integer by which the index for the sequence is divided, effectively the number of times the sequence generates the same sequence item value before incrementing to the next sequence item value. <br />Example:  To generate the sequence (1; 1; 1; 2; 2; 2; 3; 3; 3; ...) the denominator is 3.<br />The use of the denominator is to allow multiple generated sequences to periodically scan a multidimensional space. For example, an (abstract) TV screen uses 2 such generators for the columns and rows of pixels. For instance, if there are 200 scan lines and 320 raster colunmns, the column-generator would have denominator 1 and the line-generator would have denominator 320.<br />
