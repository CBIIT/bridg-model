<p>Coded data in its simplest form, where only the code is not predetermined. <br/></p><p><br/></p><p>The code system and code system version are implied and fixed by the context in which the CS value occurs.  <br/></p><p><br/></p><p>Due to its highly restricted functionality, CS SHALL only be used for simple structural attributes with highly controlled and stable terminologies where:<br/></p><p><ul><li>all codes come from a single code system</li><br/></p><p><li>codes are not reused if their concept is deprecated</li><br/></p><p><li>the publication and extensibility properties of the code system are well described and understood.</li><br/></p><p></ul><br/></p>

Attribute Details
public String
  code		 Range:0 to 1
Notes: The plain code symbol defined by the code system. if the code value is empty or null, then there is no code in the code system that represents the concept.<br />Code SHALL only contain characters that are either a letter, a digit, or one of '.', '-', '_' or ':'. Code systems that are used with CS SHALL NOT define code symbols or expression syntaxes that contain whitespace or any other characters not in this list.<br />
Operation Details

codeSystem():Uid 		Sequential
Notes: Although a CS does not carry a codeSystem as an attribute, there must always be an applicable codeSystem specified by the context in which CS is used. This operation returns the codeSystem that is specified by the context. <br /><br />This operation must always return a valid codeSystem, whether or not the CS is nullFlavored, as it is taken from the context of use.
