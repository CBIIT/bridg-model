Notes: A unique symbol that establishes identity of the regulatory assessment.<br /><br />For example, NDA number, IND number, BLA, PMA, 510K, etc.
public CD
  assessmentCode		 Map:CTGOV=Expanded Access Status
Notes: A coded value specifying the regulatory designation made by the regulatory authority.<br /><br />For example, for regular submissions the code can either be approved, not approvable, approvable, complete response or cleared.<br /><br />For example, for expanded access submissions the code can be Available, No longer available, Temporarily not available, or Approved for marketing.<br /><br /><br />NOTE: For some submissions, there are business processes that will make "default" action based on timelines --i.e., if no action is taken, then the submission is "approved". <br /><br />NOTE: For a submission, there may be multiple regulatory assessments that correspond to the state transitions for a submission, but only one regulatory assessment is true at a given time.  A submission can first be "approvable" and then when the data is complete, a new regulatory assessment can be made that is "approved".
public TS.DATETIME
  assessmentDate		 Map:CTGOV=Expanded Access Status
Notes: The date (and time) on which this particular assessment is completed.<br /><br />Note: Must be at least a full Date (8 digits) but could contain a time (14 digits).
